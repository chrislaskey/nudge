Rules

  • Matching Pattern - The matching pattern is a regular expression used to match a requested host name to a specific rule. For example the simple pattern "nytimes" will match nytimes.com, www.nytimes.com, blogs.nytimes.com and so on. See the examples below for more advanced examples, and use the URL Pattern testing tool below to verify your patterns.
  • Daily Visit Limit - The number of visits allowed in a 24 hour period. This resets every day at midnight.
  • Grace Period - optional The number of minutes in which a repeat visit will not count against the daily visit limit. This is optional but highly recommended, otherwise any attempt to click on links after the first page loads will fail.
  • Cooldown Period - optional The number of minutes before a new visit is allowed.
